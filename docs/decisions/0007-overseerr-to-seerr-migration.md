# 0007 — Migrated overseerr → Seerr

> **Status:** completed 2026-04-26. Inventory + compose template flipped to
> Seerr; Seerr's first-boot auto-migration ran cleanly; UI, API, and
> Plex/Sonarr/Radarr connectivity all verified post-deploy.

## Context

The `lscr.io/linuxserver/overseerr` image we deployed tracked the original
`sct/overseerr` codebase, which was archived 2026-02-15 in favor of
**Seerr** — a unified project that merged Overseerr and Jellyseerr.
Continuing on the archived fork meant no security fixes, no Plex/TMDb API
drift fixes, and no new features.

The user's overseerr instance is a **stateful** service: requests, users,
mapped Plex libraries, and Sonarr/Radarr connections all live in
`/home/caleb/config/overseerr/db/db.sqlite3` and `settings.json`. A naive
image swap was unlikely to work because Seerr's schema and settings format
diverged from Overseerr's during the merger.

## Decision

Migrate to the official Seerr image as a single, scheduled cutover.
Reading https://docs.seerr.dev/migration-guide on 2026-04-26 confirmed:

- **Migration is automatic.** Mounting the existing Overseerr appdata at
  Seerr's config path (`/app/config`) triggers an in-place migration of
  both `settings.json` and `db.sqlite3` on first boot.
- **No LSIO image exists** — the official `ghcr.io/seerr-team/seerr` is
  the only supported image.
- **Internal port is 5055** (unchanged from Overseerr).
- **Config path changed**: Seerr uses `/app/config`; LSIO Overseerr used
  `/config`. The compose template's volume target had to change.
- **`init: true`** is required by Seerr's compose example.
- Container runs as **UID 1000**, which already matches our `puid`.

We chose to keep the inventory service name `overseerr` (and therefore
`request.caleb.trade`, the appdata path `/home/caleb/config/overseerr`,
and the compose project name) to keep the migration small. Renaming to
`seerr` is a follow-up change if/when desired — it would require a
matching `cloudflared tunnel route dns` per decisions/0004 plus an
appdata move.

## Settings ownership: why the template was deleted, not updated

The post-migration `/app/config/settings.json` contains substantial
state that Seerr generates and manages itself:

- `clientId`, `vapidPrivate`, `vapidPublic` — push notification keypair
- `sessionSecret` — invalidates active sessions if rewritten
- `apiKey` — **regenerated by Seerr during migration** (new format:
  base64 of `<unix-millis-of-migration> <uuid>`). Our SOPS-stored
  `overseerr_secret.api_key` (old hex form) is now stale.
- `notifications.*` — ~1k lines of agent config (email, discord,
  webhooks, ntfy, etc.) that Seerr writes from the UI

Trying to template this on top of Seerr's runtime-managed values is a
recipe for breaking sessions, push notifications, and auth. We instead
follow the same pattern Plex uses: **don't template the settings file;
push deltas via the API at deploy time** (see
`api_wiring.yml`'s overseerr section, which calls `/api/v1/settings/...`
endpoints to enable auto-tagging, import Plex users, etc.).

DR for settings is covered by the nightly restic backup (decisions/0009).

## Why

The archived overseerr would accumulate untriaged bugs and unpatched
dependencies. The user has explicitly chosen to follow upstream rather
than maintain a fork.

## Outstanding follow-up

- **`overseerr_secret.api_key` in SOPS is stale** (Seerr regenerated
  during migration). The api_wiring.yml provisioning tasks that POST to
  `/api/v1/settings/...` are silently no-ops until SOPS is refreshed.
  Update via `make edit-secrets`, pulling the new value from the
  Seerr UI (Settings → General → API Key) or the live settings.json.
- **api_wiring.yml endpoints**: still use Overseerr's `/api/v1/...` paths.
  Verify each `uri` task on the running Seerr instance once the API key
  is refreshed, in case Seerr renamed routes.

## What breaks if you skip the migration plan

- **Lost request history** if the auto-migration is interrupted (e.g. by
  re-templating settings.json over the migrated file). Restic snapshot
  `368194e7` is the rollback path.
- **Broken Sonarr/Radarr/Plex wiring** if `api_wiring.yml`'s endpoints
  point at routes Seerr renamed. Watch for silent skips after deploy.
- **Public DNS gap** if a future inventory rename happens without a
  matching cloudflared route refresh — see decision 0004.
